# Research Framework

Use this framework to make recommendations that improve YouTube and Patreon traffic quality, not just raw clicks.

## 1) Clarify Intent

Define the visitor intent before proposing design changes:
- Learn from free videos (YouTube-first intent)
- Support creator directly (Patreon-first intent)
- Compare both options before deciding

Map each page section to one intent. Avoid mixed messaging in the same block.

## 2) Audit Messaging Hierarchy

Check the first screen and first scroll:
- State one clear value promise in the hero.
- Explain who the content is for.
- Show the next action within 3-5 seconds of reading.

Preferred CTA pair:
- Primary: "Watch on YouTube"
- Secondary: "Support on Patreon"

Use section-level CTA variants for experiments:
- "Watch the latest tutorial"
- "Join for bonus breakdowns"

## 3) Build a Simple Funnel Model

Use three stages:
1. Discover: Visitor arrives from search/social.
2. Trust: Visitor consumes proof (preview video, testimonials, examples).
3. Convert: Visitor chooses YouTube or Patreon action.

For each stage, identify one friction point and one fix.

## 4) Recommend by Impact x Effort

Prioritize recommendations using this order:
1. High impact, low effort
2. High impact, medium effort
3. Medium impact, low effort

Avoid low-impact polish before CTA clarity and proof blocks are fixed.

## 5) Demand Measurement

Require measurable outcomes for each recommendation:
- CTA click-through rate (YouTube, Patreon)
- Engagement depth (time on page, scroll depth)
- Conversion intent (outbound click events by source block)

Mark assumptions explicitly if analytics access is missing.
